ACTIVIDAD #5

MITRE ATT&CK

  • Descripción breve: Marco que documenta técnicas reales usadas por atacantes.
  • Fases de implementación: Basado en tácticas y técnicas (TTPs), no fases formales.
  • Objetivo principal: Mejorar detección y respuesta ante ataques.
  • Escenarios de uso: SOC, Blue Team, Red Team, monitoreo de amenazas.
  • Orientación: Defensa / Evaluación
  • Autor / Organismo responsable: MITRE Corporation
  • URL de material: https://attack.mitre.org
  • Certificaciones: No directa
  • Versiones vigentes: Actualizaciones frecuentes

OWASP WSTG

  • Descripción breve: Guía para evaluar vulnerabilidades en aplicaciones web.
  • Fases de implementación: Revisión, análisis, pruebas manuales, explotación y reporte.
  • Objetivo principal: Detectar fallas en aplicaciones web.
  • Escenarios de uso: Bancos, fintech, e-commerce.
  • Orientación: Evaluación / Ataque ético
  • Autor / Organismo responsable: OWASP Foundation
  • URL de material: https://owasp.org/www-project-web-security-testing-guide/
  • Certificaciones: Base para eWPT, OSCP
  • Versiones vigentes: Versión 4.x activa

NIST SP 800-115

  • Descripción breve: Norma formal para auditorías de seguridad.
  • Fases de implementación: Planeación, descubrimiento, pruebas, reporte.
  • Objetivo principal: Cumplimiento y control de riesgos.
  • Escenarios de uso: Gobierno, corporativos, compliance.
  • Orientación: Evaluación / Defensa
  • Autor / Organismo responsable: National Institute of Standards and Technology
  • URL de material: https://nvlpubs.nist.gov/nistpubs/legacy/sp/nistspecialpublication800-115.pdf
  • Certificaciones: Base para CISSP, CISA
  • Versiones vigentes: Vigente como referencia

OSSTMM

  • Descripción breve: Metodología integral para redes, procesos y seguridad física.
  • Fases de implementación: Análisis de vectores, métricas, validación y reporte.
  • Objetivo principal: Medir nivel real de seguridad operativa.
  • Escenarios de uso: Auditorías completas empresariales.
  • Orientación: Evaluación / Auditoría
  • Autor / Organismo responsable: Institute for Security and Open Methodologies
  • URL de material: https://www.isecom.org/OSSTMM.3.pdf
  • Certificaciones: N/A (oficialmente)
  • Versiones vigentes: Versión 3.x

PTES

  • Descripción breve: Estándar práctico para pruebas de intrusión profesionales.
  • Fases de implementación: Acuerdos, reconocimiento, explotación, post-explotación, informe.
  • Objetivo principal: Ejecutar pentesting profesional.
  • Escenarios de uso: Consultoras, ethical hacking.
  • Orientación: Ataque / Evaluación
  • Autor / Organismo responsable: Comunidad PTES
  • URL de material: http://www.pentest-standard.org/
  • Certificaciones: Base para OSCP, CEH
  • Versiones vigentes: Estándar activo

ISSAF

  • Descripción breve: Marco antiguo para evaluaciones técnicas y organizacionales.
  • Fases de implementación: Recopilación, mapeo, explotación, análisis.
  • Objetivo principal: Evaluar infraestructura y procesos.
  • Escenarios de uso: Auditorías tradicionales.
  • Orientación: Evaluación
  • Autor / Organismo responsable: Open Information Systems Security Group
  • URL de material: https://bibdigital.epn.edu.ec/bitstream/15000/16740/1/CD-7336.pdf
  • Certificaciones: N/A
  • Versiones vigentes: Poca actualización

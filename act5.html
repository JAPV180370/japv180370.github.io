ACTIVIDAD #6

Las redes privadas virtuales (VPN) Site-to-Site permiten interconectar dos redes geográficamente separadas a través de un medio no confiable como Internet, garantizando confidencialidad, integridad y autenticación mediante protocolos criptográficos como IPSec.

En esta práctica se configuró un túnel IPSec entre dos routers, simulando un escenario empresarial donde dos sucursales requieren comunicación segura sobre infraestructura pública.

Propósito de la práctica

  • Configurar un túnel VPN Site-to-Site utilizando IPSec.
  • Permitir comunicación segura entre dos redes remotas.
  • Proteger la información mediante cifrado y autenticación.

Arquitectura de red implementada

  • Dispositivos involucrados: R1 – ISP – R3.
  • El router ISP simula la red pública (Internet).
  • Red LAN 1: 192.168.1.0/24.
  • Red LAN 2: 192.168.3.0/24.

Representación lógica simplificada: 

							  LAN 1 (192.168.1.0/24)
									  |
									[R1]
									  |
									(ISP)
									  |
									[R3]
									  |
							  LAN 2 (192.168.3.0/24)

El tráfico entre ambas LAN viaja cifrado a través del túnel IPSec.

Activación de capacidades criptográficas

  • Se habilitó el paquete securityk9.
  • Se activaron funciones criptográficas necesarias.
  • Se reinició el equipo para aplicar cambios.

Este paso es indispensable en routers Cisco que requieren licencia para funciones de cifrado.

Configuración inicial de conectividad

  • Configuración de interfaces LAN y WAN.
  • Asignación de direcciones IP.
  • Definición de rutas estáticas.

Antes de implementar IPSec, es obligatorio confirmar conectividad entre extremos WAN.

Configuración criptográfica en R1

  • Creación de política ISAKMP (Fase 1).
  • Definición de clave precompartida.
  • Configuración de transform-set (Fase 2).
  • Creación y aplicación de crypto map.
  • Definición de ACL para tráfico interesante.

La Fase 1 establece el canal seguro de negociación y la Fase 2 protege el tráfico de datos.

Configuración espejo en R3

  • Replicación de parámetros criptográficos.
  • Ajuste de direcciones IP remotas.
  • Validación de coincidencia de políticas.
  • Aplicación del crypto map en interfaz WAN.

La correcta coincidencia de parámetros es crítica para el establecimiento del túnel.

Validación del túnel IPSec

  • Comando: show crypto isakmp sa
  • Comando: show crypto ipsec sa
  • Verificación de estado QM_IDLE.
  • Pruebas de conectividad entre LANs.

El estado QM_IDLE indica que la negociación fue exitosa y el túnel está activo.

Resultados observados

  • Comunicación bidireccional establecida.
  • Tráfico cifrado correctamente.
  • Túnel IPSec operativo.
  • Sin pérdida de paquetes en pruebas realizadas.

Conclusión

La implementación del túnel VPN Site-to-Site permitió comprender de forma práctica el funcionamiento de IPSec, incluyendo la negociación ISAKMP (Fase 1) y la protección de datos en Fase 2. Este tipo de configuración es ampliamente utilizado en empresas para conectar sucursales de manera segura sin depender de enlaces dedicados.

En entornos reales, una mala configuración de parámetros criptográficos o ACL puede impedir el establecimiento del túnel o dejar expuesto tráfico sensible. Por ello, es fundamental validar cada etapa, documentar configuraciones y aplicar buenas prácticas como el uso de algoritmos robustos y autenticación fuerte.

Buenas prácticas adicionales

  • Utilizar contraseñas robustas o certificados digitales.
  • Respaldar configuraciones periódicamente.
  • Documentar cambios en bitácora técnica.
  • Realizar pruebas de continuidad operativa.
  • Actualizar firmware y parches de seguridad.
